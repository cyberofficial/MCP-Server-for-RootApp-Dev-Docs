<header><h1>Key-value store persistence</h1></header>
<p>The <a class="" href="/docs/bot-api-reference/type-aliases/KeyValueStore/">KeyValueStore</a> type stores your data as key-value pairs. Data is stored in a SQLite file so it will persist across restarts of your server. Root automatically backs up and restores the data file so you won't lose data if your server restarts unexpectedly.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-use-it">How to use it<a href="#how-to-use-it" class="hash-link" aria-label="Direct link to How to use it" title="Direct link to How to use it" translate="no">​</a></h2>
<p>Root automatically creates an instance of the <a class="" href="/docs/bot-api-reference/type-aliases/KeyValueStore/">KeyValueStore</a> type for you and makes it available through the <code>rootServer.dataStore.appData</code> property.</p>
<p>Here are the key details to keep in mind:</p>
<ul>
<li class="">Keys are always strings</li>
<li class="">Values can be any type (Root saves them as JSON strings internally)</li>
<li class="">All method calls are asynchronous.</li>
</ul>
