<h1>Community log | Root developer documentation</h1>
<header><h1>Community log</h1></header>
<p>Use the <strong>community log</strong> to send messages from your server to the administrators of a Root community. It's a good way to report important events, warnings, or errors that need human attention.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="what-is-the-community-log">What is the community log?<a href="#what-is-the-community-log" class="hash-link" aria-label="Direct link to What is the community log?" title="Direct link to What is the community log?" translate="no">​</a></h2>
<p>The community log is a persistent, admin-visible log. It appears inside the Root client and helps the community understand what's happening in your code. Each community has its own log.</p>
<ul>
<li class="">The API is available from your <strong>server-side code</strong></li>
<li class="">The log is visible to community members with the <strong>ChannelManageApp</strong> permission</li>
<li class="">Messages are not visible to you, the developer. They're displayed inside the <strong>Root client</strong>, not the <a href="https://dev.rootapp.com" target="_blank" rel="noopener noreferrer" class="">Root Developer Portal</a></li>
<li class="">Messages are <strong>retained for 90 days</strong>.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="when-to-use-it">When to use it<a href="#when-to-use-it" class="hash-link" aria-label="Direct link to When to use it" title="Direct link to When to use it" translate="no">​</a></h2>
<p>Use the community log to report things that should be seen by a human administrator. For example:</p>
<ul>
<li class="">A user tried to post something invalid</li>
<li class="">Your server hit an unexpected condition but recovered</li>
<li class="">Your server hit an unexpected condition and had to shut down.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-write-to-the-community-log">How to write to the community log<a href="#how-to-write-to-the-community-log" class="hash-link" aria-label="Direct link to How to write to the community log" title="Direct link to How to write to the community log" translate="no">​</a></h2>
<p>To log a message, call the <code>create</code> method on <code>rootServer.dataStore.logs.community</code>. You pass a request object that contains the log message and its severity. The returned response object contains the unique ID for the log entry you just created (there's currently nothing to do with this ID, it's reserved for future use).</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="message-levels">Message levels<a href="#message-levels" class="hash-link" aria-label="Direct link to Message levels" title="Direct link to Message levels" translate="no">​</a></h3>
<p>Use the appropriate <code>communityAppLogType</code> to indicate severity:</p>
<table><thead><tr><th>Type</th><th>Description</th></tr></thead><tbody><tr><td><code>Info</code></td><td>General information</td></tr><tr><td><code>Warn</code></td><td>A warning that something might be wrong</td></tr><tr><td><code>Error</code></td><td>Something failed, but the system can continue</td></tr><tr><td><code>Fatal</code></td><td>A critical failure that requires a code update</td></tr></tbody></table>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="permissions">Permissions<a href="#permissions" class="hash-link" aria-label="Direct link to Permissions" title="Direct link to Permissions" translate="no">​</a></h2>
<p>Only members with the <code>ChannelManageApp</code> permission in the community will see your messages. Make sure you only log things that administrators need to know.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="summary">Summary<a href="#summary" class="hash-link" aria-label="Direct link to Summary" title="Direct link to Summary" translate="no">​</a></h2>
<ul>
<li class="">Use <code>rootServer.dataStore.logs.community.create()</code> to write to the community log</li>
<li class="">Messages are shown to community members with the <code>Manage Apps</code> permission inside the Root client</li>
<li class="">Messages are kept for 90 days</li>
</ul>
<p>The community log is your conduit directly to the community. Use it to keep them informed.</p>
