<h1>Bot project overview | Root developer documentation</h1>
<header><h1>Bot project overview</h1></header>
<p>This section covers how to set up and work with a Root Bot. You'll see how the project is structured, what tools it uses, and how to run it locally for testing.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="bot-coding-options">Bot coding options<a href="#bot-coding-options" class="hash-link" aria-label="Direct link to Bot coding options" title="Direct link to Bot coding options" translate="no">​</a></h2>
<p>Root Bots are written in TypeScript. You can pull in third-party packages from registries like <a href="https://www.npmjs.com/" target="_blank" rel="noopener noreferrer" class="">npm</a>.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="source-code-folders-and-files">Source-code folders and files<a href="#source-code-folders-and-files" class="hash-link" aria-label="Direct link to Source-code folders and files" title="Direct link to Source-code folders and files" translate="no">​</a></h2>
<p>This diagram shows the typical structure of a Root Bot. The <code>src/</code> folder contains your Bot code. By convention, your entry point is in a file named <code>main.ts</code>.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="root-api">Root API<a href="#root-api" class="hash-link" aria-label="Direct link to Root API" title="Direct link to Root API" translate="no">​</a></h2>
<p>The Root SDK includes an API of common services to help you build your Bot. The diagram below outlines the major categories. Notice that all types are exclusively for use on the server since Bots do not have any client-side code.</p>
<!-- -->
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="runtime-environment">Runtime environment<a href="#runtime-environment" class="hash-link" aria-label="Direct link to Runtime environment" title="Direct link to Runtime environment" translate="no">​</a></h2>
<p>Root Bots run in a Node.js environment. They use <strong>CommonJS modules</strong> because they're widely supported and compatible with many existing libraries. You'll have access to most core Node.js modules (with some security limitations).</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="local-testing">Local testing<a href="#local-testing" class="hash-link" aria-label="Direct link to Local testing" title="Direct link to Local testing" translate="no">​</a></h2>
<p>Root Bots can be tested locally by running the <code>rootsdk devhost</code> tool that's part of the Root SDK. There are scripts in the Bot's <code>package.json</code> file to build and run your Bot.</p>
