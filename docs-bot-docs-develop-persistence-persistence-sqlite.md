<h1>SQLite persistence | Root developer documentation</h1>
<header><h1>SQLite persistence</h1></header>
<p>Use a <strong>SQLite</strong> database to store your structured data persistently.</p>
<p>Root supports your use of a <strong>SQLite</strong> database:</p>
<ul>
<li class="">Providing the database filename.</li>
<li class="">Backing up and restoring the database file automatically so you won't lose data if your server restarts.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-use-it">How to use it<a href="#how-to-use-it" class="hash-link" aria-label="Direct link to How to use it" title="Direct link to How to use it" translate="no">​</a></h2>
<p>You need to add a few dependencies to your <code>package.json</code> file:</p>
<ul>
<li class="">Dependency on <code>sqlite3</code> to get the database engine into your server code.</li>
<li class="">Dependency on your choice of access libraries.</li>
</ul>
<p>In addition, you use the Root-provided <a class="" href="/docs/bot-api-reference/type-aliases/RootDatabaseConfig/#sqlite3filename">database filename</a> at: <code>rootServer.dataStore.config.sqlite3.filename</code> in your initialization code. You must use this filename, so Root knows where your data is stored and can do automated backups for you.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="library-options-for-sqlite-access">Library options for SQLite access<a href="#library-options-for-sqlite-access" class="hash-link" aria-label="Direct link to Library options for SQLite access" title="Direct link to Library options for SQLite access" translate="no">​</a></h2>
<p>You can use <strong>any library you prefer</strong> to access SQLite. There are three main types of libraries:</p>
<table><thead><tr><th>Category</th><th>What it does</th><th>Examples</th></tr></thead><tbody><tr><td><strong>Direct libraries</strong></td><td>Execute raw SQL statements yourself.</td><td><code>sqlite3</code>, <code>sqlite</code></td></tr><tr><td><strong>Query builders</strong></td><td>Build SQL queries programmatically.</td><td><code>Knex</code>, <code>Kysely</code></td></tr><tr><td><strong>ORMs</strong></td><td>Work with objects; the library generates SQL for you.</td><td><code>Prisma</code>, <code>TypeORM</code></td></tr></tbody></table>
<p>Root recommends you build an upload package early during your development to test how large your preferred database-access library is; some of them add quite a bit of content to your code.</p>
