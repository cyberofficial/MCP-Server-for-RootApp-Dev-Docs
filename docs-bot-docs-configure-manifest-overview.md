<h1>Manifest overview | Root developer documentation</h1>
<header><h1>Manifest overview</h1></header>
<p>Your <strong>manifest file</strong> defines how your code integrates with the Root platform. Your manifest includes your metadata, deployment instructions, community-configurable settings, and permissions. Root reads the manifest, validates it, and uses it to determine how to package and run your code.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="name-and-location">Name and location<a href="#name-and-location" class="hash-link" aria-label="Direct link to Name and location" title="Direct link to Name and location" translate="no">​</a></h2>
<p>Your manifest must be <strong>named</strong> <code>root-manifest.json</code>. The file must be located directly in your <strong>project folder</strong> so the Root SDK tooling can find it.</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="what-it-captures">What it captures<a href="#what-it-captures" class="hash-link" aria-label="Direct link to What it captures" title="Direct link to What it captures" translate="no">​</a></h2>
<ul>
<li class="">
<p><strong>Identity and version</strong>
A unique ID and a semantic version so Root can track, publish, and update your code safely.</p>
</li>
<li class="">
<p><strong>How to run your code</strong>
Where the compiled code lives and how to launch the server. Root uses this to package, deploy, and start your code.</p>
</li>
<li class="">
<p><strong>Community settings surface</strong>
Settings that community members with the <code>Manage Apps</code> permission can see and change. This lets Root generate a settings UI without you writing forms.</p>
</li>
<li class="">
<p><strong>Required permissions</strong>
The access your code needs in order to run correctly. Root uses this to ask admins for consent during installation and to enforce access at runtime.</p>
</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-root-uses-it">How Root uses it<a href="#how-root-uses-it" class="hash-link" aria-label="Direct link to How Root uses it" title="Direct link to How Root uses it" translate="no">​</a></h2>
<ul>
<li class="">
<p><strong>During publishing</strong>
Validates structure, checks versioning, and prepares the bundle based on the paths you declare.</p>
</li>
<li class="">
<p><strong>During installation</strong>
Shows permissions and settings derived from the manifest so admins know what they are allowing.</p>
</li>
<li class="">
<p><strong>At runtime</strong>
Determines how to launch your code and gates Root API calls based on the permissions you requested.</p>
</li>
</ul>
